#### 용어 정리
    예외 (Exception)
        : 개발자가 구현한 코드에서 발생, 발생할 상활을 미리 예측해 처리할 수 있도록 만든 안전 장치
          문법 오류로 인해 발생 X

    ICMP (Internet Control Message Protocol)
        : TCP/IP 계층 구조에서 IP 동작 과정에서의 전송 오류가 발생하는 경우에 대비해 오류 정보를 전송하는 목적으로 사용하는 프로토콜

    ARP (Address Resolution Protocol)
        : IP 네트워크상 IP 주소를 MAC 주소 (물리 주소)로 변환하는 프로토콜

    PPP (Point-to-Point Protocol)
        : 네트워크 분야에서 두 통신 노드 간 직접적인 연결 위해 일반적으로 사용되는 데이터 링크 프로토콜

    최적 적합 (Best Fit)
        : 가용 공간 중 가장 크기 비슷한 공간 선택해 프로세스 적재하는 방식
          (ex) 17KB 프로그램이 적재될 수 있는 곳은 20KB, 40KB인데, 17KB와 크기가 가장 비슷한 곳은 20KB임 / 내부 단편화 3KB
               내부 단편화
                    : 분할된 공간에 프로세스 적재 후 남은 공간

    커버로스 (Kerberos)
        : 컴퓨터 네트워크 인증 암호화 프로토콜

    커널 레벨 스레드
        : 커널이 각 스레드를 개별적으로 관리할 수 있음

    사용자 레벨 스레드
        : 커널 모드로 전환 없이 스레드 교환이 가능하므로 오버헤드 줄어듬

    내용 결합도 (Content Coupling)
        : 다른 모듈 내부에 있는 변수나 기능을 다른 모듈에서 사용하는 경우 결합도

    제어 결합도 (Control Coupling)
        : 어떤 모듈이 다른 모듈의 내부 논리 조직을 제어하기 위한 목적으로 제어 신호를 이용하여 통신하는 경우 결합도

    공통 결합도 (Common Coupling)
        : 파라미터 아닌 모듈 밖에 선언되어 있는 전역 변수 참조, 전역 변수를 갱신하는 식으로 상호작용하는 경우 결합도

    스탬프 결합도 (Stamp Coupling)
        : 모듈 간 인터페이스로 배열이나 객체, 구조 등이 전달되는 경우 결합도

    응집도
        : 모듈 독립성 나타내는 개념
          모듈 내부 구성요소 간 연관 정도

    결합도
        : 모듈과 모듈 사이 상호의존 or 연관 정도
               

#### 임계 구역 접근 제어하는 상호배제 기법
    세마포어 (Semaphore)
        - 멀티 프로그래밍 환경에서 공유 자원에 대한 접근 제한하는 방법
        - P(임계 구역 들어가긴 전 수행), V(임계 구역에서 나올 때 수행) 연산 기반 구현
        - P or V 수행 동안 프로세스는 인터럽트 당하지 않게 되어 상호배제 구현 가능
| 연산 | 코드 | 설명 |
| --- | --- | --- |
| P 연산 | P(S) : while S <= 0 do skip; <br> S := S - 1; | 최초 S 값은 1 <br> S가 0이면 1이 될 때까지 대기 <br> S를 0으로 만들어 다른 프로세스가 들어오지 못하게 함 |
| V 연산 | V(S) : S := S + 1; | 현재 상태는 S가 0 <br> S를 1로 원위치시켜 해제함 |

<span style="color:blue"> 데커 알고리즘 (Dekker Algorithm) </span>
    : flag와 turn이라는 변수로 임계영역에 들어갈 프로세스 (or 스레드) 결정하는 기법
    
<span style="color:blue"> 랩포트 알고리즘 (Lamport Algorithm) </span>
    : 프로세스에게 고유한 번호 부여하고, 번호를 기준으로 우선순위 정해 우선순위 높은 프로세스 먼저 임계 구역에 진입하도록 구현하는 기법

<span style="color:blue"> 피터슨 알고리즘 (Peterson Algorithm) </span>
    - 공유 메모리 활용해 여러 개 프로세스가 하나의 자원 함께 사용할 때 문제 발생하지 않도록 해주는 기법
    - 데커 알고리즘과 유사하지만 다른 프로세스에게 진입 기회 양보한다는 차이 있음

#### FIFO 페이지 교체 알고리즘 (First In First Out)
    시간상으로 메모리에 가장 먼저 들어온 페이지를 먼저 교체하는 알고리즘
    (ex) 3개 페이지 프레임 갖는 시스템에서 페이지 참조 순서가 1, 2, 1, 0, 4, 1, 3 일 경우 FIFO 알고리즘에 의한 페이지 교체의 경우 프레임의 최종 상태
            -> 프레임이 3개이므로 순서대로 1, 2 까지 넣고, 다음 1은 이미 할당되어 있으므로 패스
               다음 0을 세번째로 넣음 (1, 2, 0)
               4는 가장 먼저 들어온 1을 빼고 그 자리에 4를 넣음 (4, 2, 0)
               다시 1 들어가는데 두번째 들어왔던 2 빠지고 그 자리에 1 들어감 (4, 1, 0)
               마지막 3은 가장 마지막에 들어온 값인 0 자리로 채운다
                    => 4, 1, 3

```C
// C언어에서 0이 아니면 참, 0이면 거짓으로 인식하고, 계산한 결과는 참이면 1로, 거짓이면 0이 됨
#include <stdio.h>

int main(int argc, char *argv[]){
    int a = 5, b = 3, c = 12;
    int t1, t2, t3;
    t1 = a && b;
// a는 5이므로 참, b는 3이므로 참이기 때문에 (참 && 참)은 참 이므로 t1은 1

    t2 = a || b;
// a는 5이므로 참, b는 3이므로 참이기 때문에 (참 || 참)은 참 이므로 t2은 1 

    t3 = !c;
// c는 12이므로 참이지만 !(NOT)연산에 의해 참은 거짓 되므로 t3은 0

    printf("%d", t1 + t2 + t3);
    return 0;
// 실행결과는 2
}
```

#### HRN 스케줄링
    대기 중인 프로세스 중 대기시간이 긴 프로세스일 경우 우선순위가 높아지게 해 우선순위를 결정하는 비선점 스케줄링 기법

HRN 우선순위 공식 <br>

#### 유닉스 계열 운영체제 특징
***대다 사이계***

    대화식 운영체제 기능 제공
    다중 작업 (Multi-Tasking) 기능 제공
        하나 이상의 작업에 대해 백그라운드에서 수행 가능
    다중 사용자 (Multi-User) 기능 제공
    이식성 제공
        이식성 높으며 장치 간 호환성 높음
    계층적 트리 구조 파일 시스템 제공


#### UDP 프로토콜
    4계층인 전송 계층으로, 주소 지정하고, 경로 설정하는 기능은 3계층인 네트워크 계층에서 수행
    TCP와 같이 트랜스포트 계층에 존재
    UDP 특성
        비연결성 및 비신뢰성
            - 데이터그램 지향의 전송계층용 프로토콜
            - 메시지가 제대로 도착했는지 확인하지 않음
            - 검사 합을 제외한 특별한 오류 검출 및 제어 없음
        순서화되지 않은 데이터그램 서비스 제공
            : 수신된 메시지의 순서 맞추지 않음
        실시간 응용 및 멀티캐스팅 가능
            : 빠른 요청과 응답 필요한 실시간 응용에 적합
        단순 헤더
            : 헤더는 고정 크기의 8비트 (TCP는 20바이트)만 사용
              오버헤드 적음


#### 파이썬 자료형
    세트 (Set) 형
        : 중복된 원소를 허용하지 않는 집합 성질 가지고 있는 자료구조
    리스트 (List) 형
        : 크기가 가변적으로 변하는 선형리스트의 성질 가지고 있는 자료구조
    튜플 (Tuple) 형
        : 초기에 선언된 값에서 값 생성, 삭제, 수정 불가능한 형태의 자료구조
          다양한 데이터 타입들을 주어진 순서에 따라 저장할 수 있으나 저장된 내용 변경 불가능
    딕셔너리 (Dictionary) 형
        : 키와 값으로 구성된 객체를 저장하는 구조로 되어 있는 자료구조

    우연적 응집도 (Coincidental Cohesion)

    통신적 응집도

#### IP 프로토콜
    Data에 대해서는 체크섬 (Checksum)을 하지 않고, 헤더 (Header)에 대해서만 체크섬 (Checkum) 제공
    패킷을 분할, 병합하는 기능 수행하기도 함
    비연결형 서비스 제공
    Best Effort 원칙에 따른 전송 기능 제공
